# Instructions

## Technology Stack
- **Framework**: Vue 3 (Composition API with `<script setup>`).
- **Language**: TypeScript (using `vue-tsc` for type checking).
- **UI Library**: Ant Design Vue 4.
- **Icons**: Ant Design Icons (`@ant-design/icons-vue`).
- **Styling**: Tailwind CSS 4 (using `@tailwindcss/vite`).
- **Internationalization**: Custom lightweight implementation in `src/utils/i18n.ts` (JSON-based).
- **Build Tool**: Vite 8+ (configured for library mode).

## Project Structure & Goal
- **Goal**: Build a high-quality Vue component library suitable for npm distribution.
- **Library Entry**: `src/index.ts` is the primary entry point for exporting public components and functions.
- **Development Entry**: `src/main.ts` and `src/App.vue` are used for local testing and demonstrations.
- **Localization**: Translations are managed in `src/locales/` using JSON files.

## Coding Style and Component Design
- **Atomization**: Break down UI into small, focused, and reusable Vue components.
- **Single Responsibility**: Each component should handle one specific part of the UI or logic.
- **Modularity**: Avoid monolithic files; prefer a flat or nested structure in `src/components/`.
- **TypeScript**: Use strong typing for props, emits, and internal logic.
- **CSS Management**: Use Tailwind CSS 4 for styling. Styles are automatically injected into the JS bundle via `vite-plugin-css-injected-by-js`.

## Build & Publishing
- **External Dependencies**: `vue` and `ant-design-vue` are treated as peer dependencies and externalized in the build.
- **Type Checking**: Run `npm run type-check` to validate TypeScript before building.
- **Output Formats**: The build generates both ES Module (`.es.js`) and UMD (`.umd.js`) bundles in the `dist/` directory.

## Future Considerations
- **Linting & Formatting**: Implement ESLint and Prettier for code consistency.
- **Testing**: Integrate Vitest for unit testing components.
- **Documentation**: Use VitePress or similar for library documentation.
- **Type Declaration Generation**: Integrate `vite-plugin-dts` to ship types with the package.
